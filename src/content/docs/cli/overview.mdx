HEBBS provides two CLIs depending on your deployment mode:

| Binary | Mode | Description |
|---|---|---|
| **`hebbs`** | Local (vault) | Unified binary: vault management, all memory operations, daemon, panel. No server required. |
| **`hebbs-cli`** | Remote (server) | Client that connects to a running `hebbs-server` over gRPC. |

Both CLIs share the same memory commands (`remember`, `recall`, `revise`, `forget`, `prime`, `reflect`, `insights`). The `hebbs` binary adds vault-specific commands (`init`, `index`, `watch`, `rebuild`, `list`, `serve`, `panel`).

## Unified Binary (`hebbs`)

The `hebbs` binary operates locally by default. It discovers the nearest `.hebbs/` directory (or uses `--global` for `~/.hebbs/`) and runs the full HEBBS engine in-process.

```bash
# Vault commands
hebbs init                    # Create a vault
hebbs index                   # Index all markdown files
hebbs watch                   # Watch for file changes
hebbs rebuild                 # Delete index and rebuild
hebbs list --sections         # List indexed files and sections
hebbs serve                   # Start the daemon
hebbs panel                   # Open the Memory Palace

# Memory commands (local)
hebbs remember "some fact"
hebbs recall "query"
hebbs recall --all "query"    # Search project + global vaults
```

### Global Flags

| Flag | Env Var | Description |
|---|---|---|
| `--vault PATH` | `HEBBS_VAULT` | Explicit vault path |
| `--global` | | Use global brain (`~/.hebbs/`) |
| `--endpoint URL` | `HEBBS_ENDPOINT` | Switch to remote mode (connect to server) |
| `--format` | | Output format: `human`, `json`, `raw` |
| `--color` | | Color: `always`, `never`, `auto` |
| `-v` / `-vv` | | Verbose/trace logging |

### Remote Mode

When `--endpoint` is set, the `hebbs` binary acts as a remote client (like `hebbs-cli`):

```bash
hebbs --endpoint localhost:50051 --api-key "hb_..." recall "query"
```

## Remote Client (`hebbs-cli`)

The `hebbs-cli` binary connects to a running `hebbs-server` over gRPC. It supports one-shot commands and an interactive REPL.

```bash
hebbs-cli remember --entity customer-42 "Prefers email communication"
hebbs-cli recall --entity customer-42 "contact preferences"
hebbs-cli repl    # Interactive session
```

### Authentication

```bash
export HEBBS_API_KEY="hb_..."
hebbs-cli remember "hello world"
```

## Key Capabilities

- **All 9 HEBBS operations** in both CLIs
- **Multiple output formats**: human-readable tables (default), JSON for scripting, raw for piping
- **Streaming support**: `subscribe` opens a live stream
- **Multi-vault queries**: `--all` flag merges results from project and global vaults (unified binary only)
