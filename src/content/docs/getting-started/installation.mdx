## Homebrew (macOS)

The fastest way to install on macOS:

```bash
brew install hebbs-ai/tap/hebbs
```

This installs the unified `hebbs` binary (vault mode, daemon, panel, and all memory commands) plus `hebbs-server` and `hebbs-cli` for remote server mode.

Verify:

```bash
hebbs --version
```

## Quick Install (curl)

Install the latest release with a single command:

```bash
curl -sSf https://hebbs.ai/install | sh
```

This downloads `hebbs`, `hebbs-server`, `hebbs-cli`, and `hebbs-bench` to `~/.hebbs/bin`.

Add the install directory to your PATH:

```bash
export PATH="$HOME/.hebbs/bin:$PATH"
```

To make this permanent, add that line to your `~/.zshrc`, `~/.bashrc`, or `~/.profile`.

Verify the install:

```bash
hebbs --version
hebbs-server --version
hebbs-cli --version
```

## Which Binary Do I Need?

| Binary | Purpose |
|---|---|
| **`hebbs`** | Unified binary for local vault mode. Includes init, index, watch, recall, daemon, and panel. This is what most users want. |
| `hebbs-server` | Standalone gRPC/REST server for multi-tenant production deployments. |
| `hebbs-cli` | Client CLI that connects to a remote `hebbs-server`. |
| `hebbs-bench` | Benchmark runner for performance testing. |

For personal use, coding agents, and local knowledge management, you only need `hebbs`.

### Install Options

Pin a specific version:

```bash
HEBBS_VERSION=v0.1.0 curl -sSf https://hebbs.ai/install | sh
```

Change the install directory:

```bash
HEBBS_INSTALL_DIR=/usr/local/bin curl -sSf https://hebbs.ai/install | sh
```

### Supported Platforms

| Platform | Architecture | Binary |
|----------|-------------|--------|
| macOS | Apple Silicon (arm64) | `hebbs-macos-arm64` |
| Linux | x86_64 | `hebbs-linux-x86_64` |
| Linux | aarch64 | `hebbs-linux-aarch64` |

## Docker

Run HEBBS in a container:

```bash
docker run -p 6380:6380 -p 6381:6381 ghcr.io/hebbs-ai/hebbs:latest
```

With persistent storage:

```bash
docker run -p 6380:6380 -p 6381:6381 \
  -v hebbs-data:/data \
  ghcr.io/hebbs-ai/hebbs:latest
```

## Systemd (Bare Metal / VM)

On Linux servers, install HEBBS as a systemd service for automatic startup, restart-on-failure, and log management via journald:

```bash
curl -sSf https://hebbs.ai/install | sudo sh -s -- --with-systemd
```

This creates:

| Path | Purpose |
|------|---------|
| `/usr/local/bin/hebbs-server` | Server binary |
| `/etc/hebbs/hebbs.toml` | Configuration file |
| `/etc/hebbs/hebbs.env` | Environment overrides and API keys (mode 600) |
| `/var/lib/hebbs` | Data directory (owned by `hebbs` user) |
| `/etc/systemd/system/hebbs-server.service` | Systemd unit |

Enable and start the service:

```bash
sudo systemctl enable --now hebbs-server
```

Common commands:

```bash
sudo systemctl status hebbs-server    # check status
sudo systemctl stop hebbs-server      # graceful shutdown (SIGTERM)
sudo systemctl restart hebbs-server   # restart after config change
journalctl -u hebbs-server -f         # stream logs
```

Edit `/etc/hebbs/hebbs.toml` for server configuration and `/etc/hebbs/hebbs.env` for API keys and environment overrides.

## Build from Source

Prerequisites: Rust 1.75+, protoc (Protocol Buffers compiler).

```bash
git clone https://github.com/hebbs-ai/hebbs.git
cd hebbs
cargo build --release
```

Binaries are output to `target/release/`:

```bash
./target/release/hebbs-server --version
./target/release/hebbs-cli --version
```
