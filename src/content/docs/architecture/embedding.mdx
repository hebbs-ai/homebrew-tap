HEBBS generates vector embeddings locally using ONNX Runtime -- no external API calls on the hot path. The embedding engine converts memory content into dense vectors used by the HNSW index for similarity search.

## Model: EmbeddingGemma-300M (default)

HEBBS ships with [EmbeddingGemma-300M](https://huggingface.co/google/embedding-gemma-300m) as the default embedding model.

| Property | Value |
|---|---|
| Dimensions | 768 |
| Model size | ~600 MB (ONNX format, model.onnx + model.onnx_data) |
| Max sequence length | 2048 tokens |
| Language | Multilingual |
| Pooling | Mean pooling (built into ONNX graph) |

### Why EmbeddingGemma-300M

- **Higher dimensionality** -- 768 dimensions capture richer semantic relationships than smaller models.
- **Multilingual** -- supports a wide range of languages out of the box.
- **Longer context** -- 2048 token max sequence length handles larger memory content without truncation.
- **Mean pooling built in** -- the ONNX graph includes mean pooling, so no post-processing is needed.

EmbeddingGemma requires two files: `model.onnx` (graph definition) and `model.onnx_data` (external weights file). Both are downloaded automatically on first use.

## Model: BGE-small-en-v1.5 (legacy)

[BGE-small-en-v1.5](https://huggingface.co/BAAI/bge-small-en-v1.5) is available as a lightweight alternative.

| Property | Value |
|---|---|
| Dimensions | 384 |
| Model size | ~33 MB (ONNX format) |
| Max sequence length | 512 tokens |
| Language | English |
| MTEB score | 62.x (competitive with models 10x larger) |

### Why BGE-small

- **Small footprint** -- 33 MB fits comfortably in edge and containerized deployments.
- **Fast inference** -- sub-millisecond per embedding on modern hardware.
- **Strong quality** -- top-tier retrieval quality for its size class on the MTEB benchmark.
- **Permissive license** -- MIT licensed, suitable for commercial use.

## ONNX Runtime

The embedding model runs via ONNX Runtime, which provides:

- **Cross-platform inference** -- same model runs on Linux, macOS, and Windows.
- **Hardware acceleration** -- automatic dispatch to available hardware accelerators.
- **Optimized kernels** -- ONNX Runtime includes optimized BLAS kernels for matrix operations.

## Hardware Acceleration

HEBBS automatically detects and uses available hardware accelerators:

| Platform | Accelerator | Config |
|---|---|---|
| macOS (Apple Silicon) | CoreML | `acceleration = "coreml"` |
| Linux/Windows (NVIDIA) | CUDA | `acceleration = "cuda"` |
| Any | CPU (default) | `acceleration = "cpu"` |
| Auto-detect | Best available | `acceleration = "auto"` |

```toml
[embedding]
model = "embeddinggemma-300m"
acceleration = "auto"
```

## Batch Amortization

Embedding generation is the most expensive operation on the remember path. HEBBS amortizes this cost through batching:

1. **Incoming memories** are queued in a bounded buffer.
2. **Batch formation** -- when the buffer reaches `batch_size` or a timeout expires (whichever comes first), the batch is sent to the ONNX Runtime session.
3. **Parallel inference** -- the batch is processed as a single padded tensor, amortizing the per-inference overhead.

```toml
[embedding]
batch_size = 32
batch_timeout_ms = 5
```

### Latency Characteristics

| Scenario | Latency |
|---|---|
| Single embedding (CPU) | ~1-2 ms |
| Batch of 32 (CPU) | ~8-12 ms (~0.3 ms per item) |
| Single embedding (CoreML/Apple Silicon) | ~0.5 ms |
| Batch of 32 (CUDA) | ~2-4 ms (~0.1 ms per item) |

## Tokenization

The tokenizer is bundled with the model. Text is tokenized with the following behavior:

- Text longer than the model's max sequence length is truncated (with a warning metric emitted).
- Empty text produces a zero vector.
- Unicode text is normalized via NFKC before tokenization.

## Custom Models

HEBBS supports any ONNX-format embedding model that produces fixed-dimensional output. Switch models with:

```bash
hebbs config set embedding.model embeddinggemma-300m
hebbs config set embedding.model bge-small-en-v1.5
```

Or configure directly in `.hebbs/config.toml`:

```toml
[embedding]
model = "embeddinggemma-300m"
dimensions = 768
```

For fully custom models:

```toml
[embedding]
model_path = "/data/models/custom-model.onnx"
dimensions = 768
tokenizer_path = "/data/models/custom-tokenizer.json"
```

Changing the embedding model requires re-embedding all existing memories (via the `hebbs rebuild-embeddings` command).
